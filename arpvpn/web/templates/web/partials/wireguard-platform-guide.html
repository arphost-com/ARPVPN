Cross-platform VPN setup guide (both sides)

Use this checklist for both sides of the tunnel: ARPVPN (server side) and the remote endpoint (Windows, Linux, or macOS).

ARPVPN side
  1. Create or select the local WireGuard interface.
  2. Add/edit a peer and set Mode.
    • Client: end-user device.
    • Site-to-site: remote gateway/router.
  3. Set the peer tunnel IP in IPv4.
  4. For site-to-site peers, add remote LANs in Remote site subnets (CIDR, comma-separated).
  5. Enable Full tunnel to send all remote traffic through ARPVPN (AllowedIPs = 0.0.0.0/0).
  6. Enable NAT when the remote side is behind upstream NAT.
  7. Save, then download the generated peer config (or use QR for client devices).
Verification (both sides)
  1. Bring up the tunnel on the remote side.
  2. Confirm a recent handshake appears in ARPVPN.
  3. Ping tunnel IPs both directions.
  4. For site-to-site, test traffic to a host inside each remote LAN subnet.
  5. If handshake fails, check UDP listen port, firewall rules, and PersistentKeepalive = 25 for NATed peers.

  2. Place config as /etc/wireguard/wg0.conf with secure permissions.
  3. Enable and start tunnel: wg-quick@wg0.
  4. Verify with wg show.
  5. If acting as site-to-site gateway, enable net.ipv4.ip_forward=1 and allow forwarding between LAN and WireGuard interfaces.
sudo apt install -y wireguard wireguard-tools
sudo install -m 600 peer.conf /etc/wireguard/wg0.conf
sudo systemctl enable --now wg-quick@wg0
