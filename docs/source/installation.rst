Installation
============

As a systemd service
--------------------

1. Download or unpack a release archive generated by ``build.sh``.
   The archive includes both ``install.sh`` and the ``docker/`` deployment files.
2. Run the installation script from the extracted bundle:

    .. code-block:: bash

        chmod +x install.sh
        sudo ./install.sh

3. Run ARPVPN:

    .. code-block:: bash

        sudo systemctl start arpvpn.service

Using docker
------------

1. Use the ``docker/`` directory from the repository or extracted release archive.
2. Create a local ``.env`` file and set values for your host:

    .. code-block:: bash

        cp .env.example .env

   Important variables:

   * ``ARPVPN_IMAGE`` (image or tag to run)
   * ``ARPVPN_RUNTIME_USER`` (runtime user inside container, default ``arpvpn``)
   * ``ARPVPN_CONTAINER_NAME`` (container name; set unique value per stack)
   * ``ARPVPN_COOKIE_SUFFIX`` (optional cookie namespace suffix; defaults to container name)
   * ``ARPVPN_SECURE_COOKIES`` (``0`` for mixed HTTP or HTTPS, ``1`` for strict HTTPS)
   * ``ARPVPN_HTTP_PORT`` (HTTP bind port, default ``8085``)
   * ``ARPVPN_HTTPS_PORT`` (HTTPS bind port, default ``8086``)
   * ``DATA_FOLDER`` (host path mounted to ``/data``)

   The default ``ARPVPN_IMAGE=arpvpn:local`` builds and runs a local image from the bundle. The wrapper script seeds ``.env`` from ``.env.example`` automatically on first run.

3. Build and start:

    .. code-block:: bash

        ./up.sh up -d --build --force-recreate arpvpn

4. If the data folder already exists as root-owned from prior installs, fix ownership once:

    .. code-block:: bash

        sudo chown -R "$(id -u):$(id -g)" ./data

.. note::
    For side-by-side installs, use separate ``DATA_FOLDER`` paths, container names, cookie suffixes, and host ports.
